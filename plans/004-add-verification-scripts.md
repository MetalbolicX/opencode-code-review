## Steps

### Step 1: Add TypeScript and a linter as project-local devDependencies

The repo currently relies on ambient `tsc` availability. Declare the toolchain locally.

Target behavior:
- Add `typescript` as a devDependency so `tsc` is always available.
- Add one linter. Recommended default: `@biomejs/biome` (zero-config, fast, TS-native, handles both lint and format checks).
- Run `pnpm install` to update `pnpm-lock.yaml`.

**Verify**: `pnpm install` exits 0; `pnpm-lock.yaml` contains `typescript` and `@biomejs/biome`.

### Step 2: Add verification scripts to `package.json`

Replace the stub-only `scripts` block with real commands.

Target scripts to add:
- `"build": "tsc"`
- `"typecheck": "tsc --noEmit"`
- `"lint": "biome check ."`
- `"test": "echo \"Tests not yet configured — see Plan 005\" && exit 0"` (temporary until Plan 005 lands)
- `"verify": "pnpm typecheck && pnpm lint && pnpm build && pnpm test"`

Keep `@opencode-ai/plugin` at `^1.17.13`.

**Verify**: `pnpm typecheck` exits 0; `pnpm lint` exits 0; `pnpm build` exits 0.

### Step 3: Create the linter config

Create `biome.json` at the repo root with a minimal configuration.

Recommended minimal content:
- Enable recommended lint rules
- Set formatter to match existing style (2-space indent, double quotes)
- Include `src/` and exclude `dist/` and `node_modules/`

**Verify**: `pnpm lint` exits 0 with no errors.

### Step 4: Add CI gates

Update `.github/workflows/opencode-review.yml` to add deterministic checks before the AI review step.

Target: add a separate `verify` job (or steps within the existing job) that runs:
- `pnpm install --frozen-lockfile`
- `pnpm typecheck`
- `pnpm lint`
- `pnpm build`

Do NOT remove the existing AI review step. Add the deterministic gate alongside it.

**Verify**: the workflow file contains `pnpm typecheck` and `pnpm lint` steps.

### Step 5: Update README with a Development section

Add a "Development" section to `README.md` documenting:
- Required runtime: Node.js 18+ and pnpm
- Install: `pnpm install`
- Typecheck: `pnpm typecheck`
- Lint: `pnpm lint`
- Build: `pnpm build`
- Verify all: `pnpm verify`

Place it after the Installation section and before the Usage section.

**Verify**: `README.md` contains a "Development" section with the commands above.

### Step 6: Ensure `dist/` is gitignored

Check `.gitignore` for `dist/`. If missing, add it so `pnpm build` output is never committed.
